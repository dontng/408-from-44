# 408-from-44

> 从混沌到清晰。

一个人的考研 408 备考工具——真题驱动，把"做题"接上"记忆"和"诊断"，专治"看着都会、一考就废"。名字里的 **44** 是上一战的分数，也是这次的起点。

## 架构

```
studio.sh ──▶ studio.py (Python, stdlib only, 127.0.0.1:8408)
                    │
        ┌───────────┼───────────┐
        ▼           ▼           ▼
   bank/        review/      sessions/
   题目图片      state.json    每日快照
   (680题)     notes.json    (JSON + MD)
```

单文件前端 `tools/studio.html`，和纸风格三栏布局，无框架。后端仅标准库，纯文本持久化，git 同步跨机器。

## 功能

### 刷题台
- **真题题库**：2009–2025 共 17 年、680 道选择题，逐题切为单图（`bank/<年>/qNN.png`）
- **三栏布局**：左栏倒计时 + 中栏坚持热力图/进度/诊断入口 + 右栏题目流
- **答题两步**：点选项只高亮、可反复改；点「提交」才判分锁定，防止手滑。无官方答案的题自动转自判
- **历史回顾**：←/→ 方向键翻日，只读展示历史某天的完整题单和答题结果

### 遗忘曲线调度
- **Leitner 盒模型**：间隔 `[2, 4, 8, 16, 30, 45]` 天，7 次曝光在考前收齐
- **科目轮转**：每天激活两门（DS+网络 / 组成+OS），新题在两门间均衡
- **三阶段打散**：分块期（同章节聚类）→ 交错期（科目间交织）→ 全真模拟（完全随机跨科）

### 战况诊断
- **冲刺预测分**：各科正确率 × 满分求和
- **今日小结**：正确率 + 错题章节统计
- **今日处方**：基于补分价值推荐最值得练的章节
- **弱项诊断**：科目·章节按正确率排序，带"立即练"按钮
- **错题本**：所有最近做错的题，按科目章节列出题面
- **待解清单**：手动标"没懂"的题，只堆不清，不影响调度
- **未解疑问**：所有口袋里的疑问按科目分组，可浏览、切换状态

### 疑问口袋
每道题底部有一个默认折叠的口袋：

- **＋ 疑问** 点击展开，显示该题全部笔记的时间线
- 每条笔记带时间戳（`D{n}` 备考天数）、三态标签（`未解决` / `有进展` / `已解决`）
- 输入框 + 提交按钮写入笔记
- ◷ 按钮循环切换状态
- 笔记持久化在 `review/notes.json`，每次写 session 快照时同步备份
- 历史回顾页里只读展示已有笔记

数据流：
```
用户写疑问 → POST /api/notes → notes.json (git 同步)
                    ↓
            _sync_session() → session MD (副本)
                    ↓
            /api/stats → 未解疑问面板 (可集中浏览)
```

### 题图一致性
源 PDF 字号/页宽差异大，`tools/imgnorm.py` 用连通域量出汉字像素高，为每题计算显示宽度，存入 `review/imgnorm.json`。网页按此宽度渲染，屏幕汉字高恒定为 11px，不因铺满而忽大忽小。

### 工具链
| 工具 | 用途 |
|---|---|
| `slice_paper.py` | 文字层按题号坐标切题 |
| `slice_paper_ocr.py` | OCR 定位题号切题（扫描/坏编码年份） |
| `patch_q.py` | 手工补切漏题 / 跨页拼接 |
| `imgtrim.py` | 裁切题图白边 |
| `imgnorm.py` | 字号一致性 → 显示宽度 |
| `extract_answers.py` | 从答案 PDF 提取答案键 |
| `fill_missing.py` | 批量补全缺失题图 |

## 使用

```bash
bash studio.sh          # 启动刷题台 → http://127.0.0.1:8408
```

所有状态在 `review/` 下，可随仓库 git 多机同步：
- `review/state.json` — 遗忘曲线状态 + 每日进度
- `review/notes.json` — 疑问口袋笔记时间线
- `review/imgnorm.json` — 题图显示宽度（切图/裁图后需重算并重启服务）

## API

| 端点 | 方法 | 说明 |
|---|---|---|
| `/api/today` | GET | 今日完整题单（含 notes、进度、热力图、调度信息） |
| `/api/day?d=YYYY-MM-DD` | GET | 历史某天只读回顾 |
| `/api/grade` | POST | 提交答案 → 判分、更新曲线、落盘 |
| `/api/stuck` | POST | 标记/取消"没懂" |
| `/api/notes` | POST | 新增笔记 `{qid, text, status}` 或更新状态 `{qid, idx, status}` |
| `/api/stats` | GET | 战况诊断全量数据（含 unresolved_notes） |
| `/api/theme` | POST | 明暗主题偏好 |
