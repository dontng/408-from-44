- **studio 刷题台**：`bash studio.sh` 一键上线本地网页（127.0.0.1:8408，免代理）。**和室·三栏**版式(尺寸与 english2-daily 一致：`184px 340px 1fr` / gap44 / max1280 / sticky top38)：**左栏仅倒计时**；**中栏**（「坚持」热力图+近七天 + 今日进度圆点·点击跳题 + 阶段两门 + 战况诊断入口——倒计时下面的功能全在这）；**右栏**当天**全部题一条纵向流**（顶部翻日条标 `今日 · day N · 日期`，今天=day1 按日历算），按 科目·章节 分组成块，题面图裁白铺满、ABCD 小方块横排。**答题两步**：点选项只高亮、可反复改，点「提交」才判分锁定（避免手滑/瞎填即锁）。窄屏(<1120)三栏堆叠。遗忘曲线状态存 `review/state.json`（可多机同步）。无官方答案的题自动转**自判模式**，不会误判。
- **历史回顾·左右键翻日（已落地）**：刷题台顶部一左一右**手绘风翻日箭头**(双横杠箭头过 `#sketch` 滤镜抖出手绘味，hover 平移放大、点击/按键 `.press` 下沉，与 english2-daily 一致)，**键盘 ←/→** 直接翻。今天=可答的实时页(最右)，往左翻到过去任一有记录的日子，**只读**展示当天**完整题单**：答对/答错/未答三态(答错附"选了X")，侧栏小结与圆点同步成当天。
  - **题单存 `state.json` 的 `_progress[日]`**(不生成额外文件)：`roster`(当天完整题单含未答，首访 `/api/today` 时 `snapshot_roster` 落盘)+ `res`(判分时每题对错，回顾不受日后复习改判影响)。回顾 `build_day` 读之(`/api/day?d=YYYY-MM-DD`)；题图在网页直接 `<img>` 从 `bank/` 加载，无需把题存进文件。`progress_days`/`day_nav` 给日期序列与前后导航。回顾页 `renderReviewBlock` 纯只读(不带可点选项，避免误判进今天)。战况诊断视图里不响应方向键。
- **坚持热力图 + 近七天趋势（已落地）**：**中栏常驻**(三栏布局，仿 english2-daily 中栏)。GitHub 贡献图式 **SVG 热力图**(最近 5 个月滚动窗口，今天 6 月→覆盖到 2 月；周标签 Mon/Wed/Fri；格子深浅按当天**正确率**分 4 档(题量每天都≈20，没区分度，故用正确率)、悬浮出"正确率 Z% · 对/总"，今天描茜色边；少→多**渐变条**图例、右对齐到热力图右缘)+ **连续/累计坚持天数** + **近七天表**(日期·题数·正确率·↑↓趋势)。数据全由 `build_chart` 从 `_progress` 现算、随 `/api/today` 返回(`renderTrack` 渲染中栏)，零额外存储。配色沿用 english2-daily 的莫斯绿分档。翻历史日只换右栏题流，中栏总览不变。
- **战况诊断面板（已落地 · 2026-06 增强）**：中栏「战况诊断」进入(或 `#stats` 直达)；Escape 退回刷题。五项反馈顺序：①今日小结(当日正确率+答对题数+出错章节chip，0题时显示提示) ②待解清单(标"没懂"的题持久堆积，"搞懂了"撤下) ③弱项诊断(科目·章节正确率升序，最弱在前，默认显示最弱12条，可点展开全部章节) ④错题本(最近做错的题+图，显示你选了哪个选项+做错日期，多科时支持科目筛选按钮) ⑤冲刺预测分(各科正确率×满分求和，/150，目标120)。进入时"诊断中…"加载占位，数据到位后整体渲染。顶部分节锚点导航(今日小结/待解/弱项/错题/预测分)，点击平滑滚动定位。后端 `build_stats` / `/api/stats`；错题本 `mistakes` 字段含 `pick`(你选了哪个) 与 `last`(做错日期)。
- **调度器（已落地）**：遗忘曲线 + **科目轮转**（`数据结构+网络`/`组成+操作系统` 按天交替，新题两门均衡）+ **分块→交错→随机** 三阶段（`SCRAMBLE_PHASES`：8/31 前分块、11/30 前交错、之后全真模拟）。复习题(due)按曲线始终全出；新题受轮转/分块约束。无 tags 的题按题号位置兜底归科。参数全在 `studio.py` 顶部。
- **题库：17 年全部完整，680 题（2009–2025 各 40 题）** → `bank/<年>/qNN.png`，已全部**裁白**（`tools/imgtrim.py`，切题/补切时自动调用）。
- **题图字号一致化（已落地）**：源 PDF 字号/页宽差异大、裁白后每张图宽窄不一，旧版网页按宽度铺满会让屏幕字号差到一倍。`tools/imgnorm.py` 用**连通域量出的汉字像素高**(按年取中位数，即用户真正看到的字大小)算出每题**显示宽** → `review/imgnorm.json`；网页按此宽显示(不铺满)，屏幕汉字高恒为 11px(全年实测 10.1–11.0)。为容下归一化后的图(中位 621px)，版式主栏放宽到 1180/gap64。`studio.py` 仅读 json(保持纯标准库)，**dispW 在进程启动时载入——改图/重算后必须重启服务**(`pkill -f studio.py; bash studio.sh`)，光刷新网页不生效。**新切/重裁某年后须重跑** `python3 tools/imgnorm.py [年份…]`。
- **三套切题工具**（`tools/`）：
  - `slice_paper.py` — 文字层按题号坐标切题（适用文字层正常的年份）。
  - `slice_paper_ocr.py` — OCR 定位题号坐标切题（适用坏编码/纯扫描/图片页年份）。OCR 只用来找题号，题面成品仍是原始渲染图，正文 OCR 错误不影响结果。平局取靠后起点，避开"考生须知 1~5"污染。
  - `patch_q.py` — 手工补切漏题：`scan` 列某页左边距行的 y 比例，`cut` 按页+上下比例切单题，`cut2` 跨页题竖向拼接（依赖 Pillow）。

## 常用命令
```bash
bash studio.sh                                       # 启动刷题台
python3 tools/slice_paper.py     真题pdf/XXXX.pdf XXXX   # 文字层切某年
python3 tools/slice_paper_ocr.py 真题pdf/XXXX.pdf XXXX   # OCR 切某年(扫描/坏编码)
python3 tools/patch_q.py XXXX scan <页>                 # 找漏题的 y 坐标
python3 tools/patch_q.py XXXX cut  <题号> <页> <y0> <y1>          # 补切单题
python3 tools/patch_q.py XXXX cut2 <题号> <页> <y0> 1.0 <页+1> 0 <y1>  # 跨页题
python3 tools/imgnorm.py                              # 重算题图显示宽(切/裁图后跑)
```
