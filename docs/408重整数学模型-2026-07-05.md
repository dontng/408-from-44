# 408-from-44 重整数学模型

> 固定日期：2026-07-05 凌晨  
> 时间位置：距 2026-12-19 初试约 166-167 天  
> 目的：把“哪些题值得重复、每天做多少、何时引入大题、如何抗中断”固定下来，避免以后反复重新推算处境。

## 1. 总原则

项目使命不是维护完整题库，而是服务提分：

```text
44 分起点 -> 100+ / 120+
```

408 是熟练度考试。重复是必须的，但不能把所有年份、所有题目平等重复。旧模型 `2009-2025 全部选择题 × 7 遍` 会制造过大的任务债，导致项目越用越少。

新模型采用：

```text
核心年份高重复
早年年份战略降权
每日任务硬封顶
中断后不补债，只保温恢复
选择题先行，大题在选择题完成若干轮后接入
```

## 2. 选择题年份分层

| 层级 | 年份 | 题量 | 目标遍数 | 作用 |
|---|---:|---:|---:|---|
| 核心卷 | 2018-2025 | 320 | 7 遍 | 主战场，必须打穿 |
| 训练卷 | 2015-2017 | 120 | 3 遍 | 补题型、补熟练度 |
| 补洞卷 | 2013-2014 | 80 | 1 遍 | 缺口样本，弹性使用 |
| 放弃主线 | 2009-2012 | 160 | 默认 0 遍 | 不进默认计划，只在某类题持续圈外时抽用 |
| 待加入 | 2026 | 40 | 后期另算 | 公布后用于校准和模拟 |

这里的“放弃主线”不是删除题目，而是不承担默认重复任务。

## 3. 选择题曝光量

保底主线：

```text
2018-2025: 320 × 7 = 2240
2015-2017: 120 × 3 = 360
2013-2014: 80 × 1 = 80
基础曝光量 = 2680
```

考虑 `圈外 / 误选 / 心虚` 题需要额外回炉，预留约 25% 容灾与回炉预算：

```text
2680 × 1.25 ≈ 3350 次选择题答题事件
```

如果按有效学习日计算：

```text
日历剩余：约 166-167 天
有效学习日：按 130-140 天估算

3350 / 140 ≈ 24
3350 / 130 ≈ 26
```

结论：

```text
选择题正常日目标：20-26 题
选择题每日硬上限：30 题
```

超过 30 题不是进度，而是崩盘风险。

## 4. 每日任务量

每日任务采用容量模型，不采用“due 题全部清空”模型。

```text
今天最多 N 题
在 N 个坑位里挑最该出现的题
没排上的题不算欠债，只回到优先队列
```

阶段建议：

| 阶段 | 每日选择题量 | 新题 | 复习/旧错 | 目标 |
|---|---:|---:|---:|---|
| 重启期 | 8-10 | 3-4 | 5-7 | 恢复打开项目 |
| 稳定期 | 15-20 | 5-8 | 10-12 | 建立节奏 |
| 主训练期 | 20-26 | 4-8 | 16-20 | 核心卷多轮复现 |
| 高强度日 | 27-30 | 视情况 | 视情况 | 偶尔冲量，不作为默认 |
| 低效日 | 8-10 | 0 | 8-10 | 保温，不断线 |

前期新题和复习必须分流；后期新题逐渐归零，主要变成核心卷复现和高风险题维护。

## 5. 题目状态模型

选择题不是只记对错，而是推动状态升级：

| 状态 | 含义 | 调度含义 |
|---|---|---|
| 圈外 | 答案不在选项内，或思路完全跑偏 | 最高风险，短期复现 |
| 误选 | 进入选项集合，但中了干扰项 | 高风险，分析关卡点 |
| 心虚 | 答对了，但讲不清规则或排除理由 | 不能算会，需要复现 |
| 稳了 | 答对且能说明规则、关卡点、排除理由 | 降频，保手感 |

选择题训练路径：

```text
先进包围圈 -> 再冲出包围圈
```

每次重复的意义不是曝光次数 +1，而是让题目从 `圈外/误选/心虚` 升级到 `稳了`。

## 6. 容灾规则

本模型不假设 166 天每天满血学习。

中断、加班、状态差时，不能补债式滚雪球：

```text
昨天没做的题不叠加到今天
今天只按今日容量重新挑题
中断后先保温，不急着加新题
```

具体规则：

| 情况 | 处理 |
|---|---|
| 低效日 | 只做 8-10 道旧错/核心复现，不做新题 |
| 中断 1-2 天 | 恢复日不加新题，只做旧错和核心卷 |
| 中断 3 天以上 | 连续 2-3 天按重启期执行 |
| 连续 3 天做不完 | 降低每日容量，不追债 |
| 圈外题增多 | 减少新题，把容量让给规则回补 |

核心原则：

```text
断了不是补昨天，而是重新保温今天。
```

## 7. 大题接入窗口

大题不从一开始混入。前期先用选择题暴露知识点，完成若干轮后，选择题速度会被熟练度压缩，节省出的时间再用于大题。

从 2026-07-05 起算：

| 时间点 | 日期 | 含义 |
|---|---|---|
| 最早试探 | 2026-08-09 左右 | 核心选择题约 2 轮后，可轻量接触大题框架 |
| 理想正式接入 | 2026-08-23 左右 | 核心选择题约 2.5-3 轮后，大题应正式进场 |
| 最迟接入 | 2026-09-06 左右 | 再晚会压缩大题的题型覆盖、限时表达和整卷节奏训练 |

阶段安排：

```text
07-05 ~ 08-09：
  选择题先行，目标是进包围圈。

08-09 ~ 08-23：
  选择题为主，大题每 2-3 天试探一次，只认题型和框架。

08-23 ~ 09-06：
  大题正式进场，但选择题继续核心复现。

09-06 以后：
  选择题保熟练度，大题系统推进，逐步转向整卷节奏。
```

## 8. 大题原则

大题不能赌“近年不重复”，也不能把 2009-2025 全部大题一开始纳入主线。

大题策略：

```text
近年大题不丢
不机械高遍数
按题型覆盖 + 近年校准
```

大题重复的目标不是记住原题，而是缩短时间、保持质量和手感：

```text
第一遍：看懂题型和标准过程
第二遍：独立复现步骤
第三遍：限时写出框架和关键得分点
```

最终目标不是为了真的提前交卷，而是：

```text
熟练到能提前完成一遍，
把剩余时间留给检查、纠错和抗考场干扰。
```

## 9. 固定结论

本项目后续默认按以下模型推进：

```text
2018-2025 选择题：7 遍主线
2015-2017 选择题：3 遍训练
2013-2014 选择题：1 遍补洞
2009-2012 选择题：默认退出主线
2026：公布后后期加入

正常日：20-26 道选择题事件
硬上限：30 道选择题事件
低效/恢复日：8-10 道，只保温

大题：
  最早 08-09 试探
  理想 08-23 正式接入
  最迟 09-06 必须接入
```

一句话：

> 用近年核心选择题的高重复，先把知识漏洞和选择题熟练度打出来；再用熟练度节省出的时间接入大题，最终靠重复和手感给考场留出审查余量。
