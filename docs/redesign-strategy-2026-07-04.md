# 408-from-44 重构策略基线

> 记录日期：2026-07-04  
> 背景：距离考试约 167 天，在职备考，上一战 408 为 44 分。项目下一轮重构以“降低启动阻力、暴露真实漏洞、提高近年真题复现质量”为核心。

## 1. 总判断

`408-from-44` 不再按“完整题库 + 全题均匀 7 遍”的工程目标推进。

原目标的问题：

- 680 题全部机械 7 遍，任务量过重。
- 维护题库、标签、调度、快照的成本过高，挤占真正做题时间。
- 打开项目时容易感到负债，而不是轻松开始。
- 对错记录太粗，不能清晰暴露“为什么错”。

新目标：

> 真题牵引基础，用近年真题反复暴露问题；通过风险分层和包围圈机制，把题目从“做过”变成“打穿”。

## 2. 不放弃多轮真题，但放弃全题机械 7 遍

多刷真题仍然必要，原因：

1. 起点低，44 分说明漏洞密集。
2. 在职备考，遗忘严重。
3. 数二经验表明，重复刷是必要手段。
4. 目标 120+，选择题不能拉垮。

但 7 遍不再理解为：

```text
所有年份、所有题目、全部平等重复 7 次。
```

而是改成：

```text
真题体系滚动多轮；高风险题、圈外题、心虚题、近年核心题获得更多复现。
```

## 3. 年份策略：近年核心，早年降权

2026 年真题暂未公布，后期再补入。

当前年份权重：

| 层级 | 年份 | 用法 |
|---|---|---|
| 核心卷 | 2018-2025 | 必须深刷，多轮复现，优先打穿 |
| 训练卷 | 2013-2017 | 用于补题型、补章节、补重复 |
| 样本卷 | 2009-2012 | 降权，不追求全刷 7 遍，只作早期题型样本和漏洞补充 |
| 待补卷 | 2026 | 公布后加入后期模拟和校准 |

题单来源建议比例：

| 来源 | 比例 |
|---|---:|
| 2018-2025 核心题 | 60% |
| 错题 / 圈外 / 心虚题 | 30% |
| 2009-2017 补洞题 | 10% |

核心原则：

> 近年真题高频深挖，早年真题低频补洞。

## 4. 基础课策略：真题牵引基础

不再走“先完整刷完四门基础课，再开始真题”的线性路线。

原因：

- 剩余时间有限。
- 在职备考连续性差。
- 四门基础课完整刷完后，真题时间会被严重压缩。

新路线：

```text
做真题 -> 暴露漏洞 -> 回看对应基础课小节 -> 回来重做同类题
```

基础课只服务三个问题：

1. 我为什么在圈外？
2. 这道题的规则是什么？
3. 同类题下次如何进圈？

## 5. 包围圈机制

选择题训练分两阶段：

```text
先进包围圈，再冲出包围圈。
```

含义：

- 先进包围圈：至少错在 A/B/C/D 的典型干扰项里，而不是算出一个选项外答案或完全跑偏。
- 冲出包围圈：看穿出题人的关卡点，知道为什么干扰项诱人但不能选。

答题结果不再只分 `right/wrong`，而要记录层级：

| 状态 | 含义 |
|---|---|
| 圈外 | 答案不在选项内，或思路完全跑偏 |
| 误选 | 进入选项集合，但中了干扰项 |
| 心虚 | 答对了，但讲不清考点或排除理由 |
| 稳了 | 答对且能说明规则、关卡点、排除理由 |
| 反杀 | 能预测出题人会如何设置干扰项 |

项目中可先落成四个极简按钮：

```text
圈外 / 误选 / 心虚 / 稳了
```

`反杀` 可以先作为后期状态，不必第一版实现。

## 6. 错因记录：先找第一处分叉点

尤其是“圈外”题，不能简单归为粗心。

处理流程：

```text
1. 不先看完整解析。
2. 重新列出题目给定条件。
3. 写出本题适用的规则。
4. 重算或重推一次。
5. 如果仍不在选项内，标记为“圈外”。
6. 看解析时只找第一处分叉点。
```

第一处分叉点指：

> 我的思路从哪一步开始不再按题目规则或机器规则走。

这比抄完整解析更重要。

## 7. 风险分层

调度核心从单纯 Leitner 曲线转为风险驱动。

建议每题维护一个风险等级：

```text
low / medium / high / critical
```

风险来源：

- 是否圈外。
- 是否误选。
- 是否心虚答对。
- 是否有未解决笔记。
- 是否连续错。
- 是否属于弱章节。
- 最近一次是否稳定答对。

建议规则：

| 风险 | 调度策略 |
|---|---|
| low | 偶尔出现，避免浪费时间 |
| medium | 正常复习曲线 |
| high | 更短间隔，优先复现 |
| critical | 每轮优先，直到降级 |

## 8. 每天最小成功动作

项目不能要求每天高强度启动。

每天最小成功动作：

```text
打开 -> 做少量题 -> 暴露 1-3 个真实漏洞 -> 关闭
```

只要留下一个真实漏洞，就算项目当天产生价值。

建议模式：

| 模式 | 作用 |
|---|---|
| 今日少量题 | 降低启动阻力 |
| 错题拷打 | 处理最近暴露的问题 |
| 漏洞看板 | 告诉今天最该补什么 |
| 近年核心卷 | 保证 2018-2025 高频复现 |

## 9. 当前重构优先级

第一优先级：降低启动阻力。

- 默认今日题单不宜过大。
- 题单应优先包含近年核心题和高风险题。
- due 题不能无限膨胀压垮当天。

第二优先级：改造答题结果模型。

- 从 `right/wrong` 升级为 `圈外/误选/心虚/稳了`。
- 保留原始对错，但新增诊断层。
- 历史快照应记录当时选项和当时状态，不能被后续覆盖。

第三优先级：风险调度。

- 新题不再全题平权。
- 近年核心题加权。
- 圈外、心虚、未解决笔记加权。
- 早年题作为补洞题低频出现。

第四优先级：基础课回补入口。

- 每个漏洞可以关联一个“应回补小节”。
- 不追求完整课程进度，只追求真题触发后的定点修复。

## 10. 一句话原则

> 不是维护一个完美题库，而是每天更清楚地知道自己为什么错，并让高价值真题反复回来。
