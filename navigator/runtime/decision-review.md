# 决策审视｜时间助理的第二道判断

每次 runtime 给出安排，都先形成一个可反驳的初案，再做独立审视；不是把原方案换一种措辞重说。审视的职责是专找错误归因、虚假推进、遗漏代价和没有出口的安排。结论只能是保留、改写或暂缓，不能为了完整而强行给计划。

## 初案必须说明

- 已知事实与尚未证实的推断；
- 本次安排要推进的唯一对象：某个 `coach/current.md` 题号、某个 `speedrun` 验收，或明确的恢复/现实事务；
- 预计占用、停止边界，以及没完成时怎样回到题单；
- 这一步为什么比其他可选动作更值得做。

## 独立审视逐项反驳

1. **推进性**：它是否真的增加题单完成、攻略验收或次日容量？若只是整理、看资料或泛泛休息，不能算推进。
2. **证据性**：对能力、疲劳、摩擦的判断是否有当天事实或既有记录支持？没有就标未知，不把猜测写成原因。
3. **代价**：它会否挤压睡眠、吃饭、通勤或已承诺的现实事务？若会，是否有更小的任务粒度或更低价值投入可先删？
4. **可验证性**：结束后用什么事实判断有效——题目结果、决策轨迹、真实专注耗时、入睡时间或次日状态？
5. **反例**：若今晚容量比估计低、任务卡住或突发事项继续发生，计划怎样降档而不脱离题单？

审视不替用户决定加班、金钱、医疗和重要关系；它只判断这些现实条件下，余下时间怎样不被虚耗或透支。
