# runtime｜时间助理的互动日记本

这里是 dontng 输入当天现实、时间助理给出判断和对策的地方。每个有实际输入的日子是一页；当天多次对话、临时变化和结果回填都留在同一页，不把一次完整咨询拆到不同目录。

你可以像正常聊天一样写一句话、贴截图、补几个时间点，或写一段当天的情况。助理负责在同一页中提取事实、解释判断、给出具体安排，并根据后续结果修正。不是填写表格，也不要求每天都创建文件。

## 文件

- [index.md](index.md)：当前入口、正在处理的专题与最近一页日记。
- [diary/](diary/README.md)：按年/月/日归档的互动日记；一页包含输入、回应和结果。
- [cases/](cases/README.md)：只有反复出现、值得单独理解的摩擦才建立专题；专题从多篇日记中取证，不替代日记。

## 写入原则

- 日记按发生日期归档；当天没有输入就不建页。内容很少也不强行扩写。
- 助理的回应必须说明依据、当晚或当天具体安排、停止边界和未完成项的处理，不只说「休息」或「努力学习」。
- 专题必须回链到具体日记，持续区分外部限制、可调整环节与尚未证实的推断。
- 公开仓库只写为管理所必需的结论；不需要把精确行程、原始健康资料或不宜公开的事件写进去。
