# runtime｜时间助理的日常运行层

这里记录的不是完整日记，而是会改变当天或后续安排的现实：睡眠、班次、外测、身体状态、学习进展、突发事项，以及助理的处理是否有效。

你不需要按固定格式上报。可以直接说「今天外测，回家很晚」「昨晚只睡了五小时」「下班后完全不想打开题」「今晚状态比预期好」，或发截图。助理应像正常对话一样结合当天条件给出判断；只有判断会影响以后时，才压缩写入文件。

## 文件

- [current.md](current.md)：当前已知的今天—未来七天状态；只保留仍有效的信息。
- [friction-log.md](friction-log.md)：发生了什么、造成什么摩擦、如何处理、结果如何。
- [rules.md](rules.md)：经过多次记录验证后才成为默认的规则；尚未验证的只作为假设。

## 运行方式

每次新输入后，助理先判断它影响今晚、明天还是本周；再结合 `work/`、`sleep/` 和学习项目的真实状态给出一个可执行的当天安排。安排应说明：今晚最需要保护什么、唯一主事是什么、何时停止、什么明确不做，以及未完成任务如何重排。

休息也必须具体化。助理不能只说「去休息」；它应结合剩余时间和疲劳来源，说明更适合吃饭洗漱、散步、低刺激放空、睡觉，还是一项低负荷的学习保温。

公开仓库只记录必要的脱敏结论；精确行程、健康原始数据和不适合公开的个人事件不写入这里。
