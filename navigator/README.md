# 生活时间账本

这里服务于一个具体的人，而不是抽象的「效率提升」。它用已有的工作与睡眠基线理解约束，再根据每天新发生的现实，帮助作出当天更合适的安排。

## 目录结构

| 目录 | 用途 |
| --- | --- |
| `dontng.md` | 服务对象：助理在为谁服务、保护什么、能替其处理什么 |
| `sleep/` | 睡眠基线：Apple Watch 的派生数据与分析 |
| `work/` | 工作基线：班型、加班、夜班与公出形成的硬约束 |
| `runtime/` | 运行层：按日互动记录与反复摩擦专题 |
| `yangruikee/` | 委托管理研究：从一位「账号由助理管理」的创作者反推可迁移的时间管理模型 |

## 阅读顺序

1. [dontng](dontng.md) — 先理解助理服务的对象与边界
2. [runtime 总览](runtime/index.md) — 从当天互动日记进入当前管理
3. [生活节律总览](生活节律总览-2026年7月.md) — 工作、睡眠、恢复三条线的历史基线
4. [2026 年 1 至 7 月睡眠汇总](sleep/2026年1至7月睡眠汇总.md) 与 [考勤分析](work/2026年1至7月考勤分析.md) — 需要回查依据时再读

日常变化进入 [runtime/](runtime/README.md)。你可以像正常聊天一样发当天发生的事；助理负责提取事实、作出安排、记录会改变后续判断的结果，而不是要求你按表打卡。

## 原则

1. 原始健康导出不提交仓库；只保留理解节律所必需的派生数据。
2. 数据用来理解约束和选择权，不用来贴「自律/懒惰」的标签。
3. 恢复、通勤、吃饭、社交与学习同为真实生活；没有记录不意味着浪费。
4. 每条长期规则必须经实际结果验证；一次状态只影响当天，不直接变成对人的结论。
