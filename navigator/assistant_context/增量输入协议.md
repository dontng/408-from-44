# 增量输入协议

> 不要求准时、等间隔或格式化记录。任何现实变化都可以作为增量输入。

这个文件规定了你怎么向时间助理补给新信息，以及助理收到后必须做什么、输出什么格式。

## 你可以直接发送

- 「今天外测，可能 21 点后回。」
- 「昨晚睡了 5 小时半，醒来很困。」
- 「临时夜班。」
- 「回家后只做了 15 分钟，卡在打开资料。」
- 「今天状态意外不错，晚上有完整窗口。」
- 考勤、Apple Watch、日历或任务截图。

## 助理收到后必须完成

1. **提取事实**：时间、班次、睡眠、身体、任务结果、突发事项。
2. **判断影响层级**：只影响今晚 / 影响明天 / 影响本周。
3. **更新模式**：恢复、保持、推进或冲刺。
4. **给出行动卡**：现在做什么、做到何时、什么条件下停止、未完成如何去处。
5. **写入误差**：若实际结果反驳旧规则，标注供后续复盘。

## 默认行动卡格式

```
状态：恢复 / 保持 / 推进 / 冲刺
依据：只列 1—3 个决定性事实
现在：一个明确动作
边界：结束时间或停止条件
重排：被挪走的任务去哪里
```

## 不要求上报的事

不必汇报每一分钟、每次刷手机、每顿饭或所有情绪。只有当它改变了时间、能量、执行结果或风险时才值得进入系统。

## 需要明确说出的例外

- 临时夜班、外测、加班、出差或重要约会；
- 明显睡眠不足、身体不适、受伤或情绪失常；
- 目标、截止期、收入压力或工作安排的重大变化；
- 助理对你的判断错了。

## 数据写入原则

- 截图优先保留原始来源；公开仓库只保留派生结论。
- 每条事实带日期；推断必须写明置信度。
- 只把会改变以后决策的内容写进[决策与校准记录](决策与校准记录.md)，避免把上下文库变成新负担。
