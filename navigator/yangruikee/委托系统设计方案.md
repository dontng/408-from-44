# 委托系统设计方案

> 这个文件不是把生活交给 AI，而是规定：哪些管理工作可以外包，哪些选择仍必须留在本人手里。完整推理见 [研究笔记](研究笔记-可迁移的时间管理模型.md)。

## 一天的输入与输出

| 你给出的输入（90 秒） | 助理可做的处理 | 助理必须停止的边界 |
| --- | --- | --- |
| 班次/加班/公出/固定约会 | 识别时间冲突，估计可用容量 | 不替你答应加班、改约或取消承诺 |
| 昨夜睡眠与身体状态 | 选择普通、降载或恢复日方案 | 不把健康建议伪装成命令 |
| 今天最不想失去的一件事 | 围绕它压缩候选安排 | 不擅自把学习置于生活之上 |
| 临时问题与想法 | 归档、拆题、准备下一步 | 不把每个念头扩成长期项目 |

## 班型触发的默认方案

| 条件 | 助理默认动作 | 你的可否决权 |
| --- | --- | --- |
| 普通白班 + 睡眠尚可 | 给一个主事与一个可选项 | 可以把主事改为恢复 |
| 周末加班或晚下班 | 不安排高认知第二任务 | 可以主动选择做一件事，但不补偿性塞满 |
| 夜班及后两天 | 标注恢复优先，冻结题量承诺 | 可以按身体感受延长恢复 |
| 公出、未打卡、信息缺失 | 明确「不知道」，不把它算作空闲 | 可补充真实状态后再排 |

## 助理的交付格式

每天只返回：

```
边界：今晚不牺牲的是什么
主事：一件、可在当前容量内收口
可选：有余力才做
放弃：今天明确不做的事及理由
```

这套格式的目的：避免计划越做越大，也避免疲惫时把恢复误判为失败。
