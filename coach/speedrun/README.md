# speedrun｜408 的决策攻略

这里不是课程、题解集合或打卡清单。它把已由题目证实的机制压成考场入口：题干出现什么条件，先抓什么，怎样直接判断或排除干扰项。题单仍由 `today/` 和 `current.md` 运行，完成题量也仍是唯一进度来源。

状态只有两种：`candidate` 表示已有一个可检验的假设；`verified` 表示至少经过两条题证或一题加改条件检验，能迁移而非记住原题。候选页不可被当作已完成覆盖，也不可替代真实作答。

每页必须有：适用边界、最小前提、决策动作、干扰项机制、证据与验收。它分别链接 [knowledge](../knowledge/README.md) 的原理和 [analysis](../analysis/README.md) 的原题；不复制正文。
