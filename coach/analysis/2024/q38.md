# 2024-38

按 408 对 TCP 慢开始的默认口径，连接建立时初始拥塞窗口为 `1 MSS`。H 发送的 3000 B 报文被分成 3 个 1000 B 的报文段：第 1 轮发送 1 段，收到确认后拥塞窗口增至 `2 MSS`，第 2 轮发送剩余 2 段，因此数据传输需要 `2 RTT`。

从 H 请求建立连接到能够发送数据，需要 `1 RTT`。数据传完后 H 主动关闭；服务器可连续发出对 FIN 的 ACK 和自己的 FIN，H 最早再经过 `1 RTT` 收到 FIN、回送最后一个 ACK，并进入 TIME_WAIT。主动关闭方不能立即进入 CLOSED，还要等待 `2 MSL`。

```text
总时间 = 建连 + 传数据 + 进入 TIME_WAIT + 等待结束
       = 1 RTT + 2 RTT + 1 RTT + 2 MSL
       = 4×10 ms + 2×30 s
       = 60.04 s
```

所以选 D。A、B 都只留出了 `1 MSL`，先排除；C 已计入 `2 MSL`，但少了 `1 RTT`，相当于没有按初始 `cwnd=1 MSS` 分两轮发送 3 个报文段。完整计时边界见 [TCP：慢开始、连接建立与主动关闭计时](../../knowledge/network/04-transport/tcp-lifecycle-timing.md)。
