# NAT：地址转换、端口转换与 UDP 校验和

## NAT 在出网时做什么

内网地址 `192.168.1.5/24` 不能在 Internet 中路由。NAT 路由器向外转发其 UDP 数据报时，必须把 IP 首部中的**源 IP 地址**改为自己的公网地址；返回报文到达路由器后，再按转换表改回内网主机的地址。

若多台内网主机共用一个公网地址，转换表还要用传输层的端口号区分会话。于是出网方向常见的对应关系是：

```text
192.168.1.5 : 49152  ──NAT──>  公网地址 : 40001
```

这里改的是 UDP 的**源端口号**。目的端仍是 Internet 上服务器监听的端口，不能随意改动，否则报文不会送到原本要访问的服务。

## 哪些字段不因 NAT 而改变

NAT 是改写已有字段，不是在 UDP 报文段前后增添字节。因此 UDP 报文段本身的**总长度不变**。IP 首部中的总长度也不因“地址从私网变公网”而改变。

不能把“经过路由器”想成“封装一层，所以 UDP 变长”。链路层帧头会随每一跳重新封装，但它不属于 UDP 报文段，更不会改变 UDP 首部的长度字段。

## 为什么 UDP 校验和必须改

UDP 校验和覆盖两部分：

1. UDP 首部和数据；
2. 由 IP 首部提供的伪首部，其中含源 IP 地址、目的 IP 地址、协议号和 UDP 长度。

NAT 改写了源端口号；即使只改源 IP 地址，伪首部也已经改变。因此旧校验和不再对应当前报文，路由器必须重算或按一补码规则增量更新 UDP 校验和。

判断 NAT 选择题时，可以按这个顺序问：

```text
需要让外网回包找到哪台内网主机？ → 源 IP / 常见为源端口
改动是否增添 UDP 字节？             → 否，总长度不变
校验范围中是否含被改字段？           → 是，校验和必须更新
```
