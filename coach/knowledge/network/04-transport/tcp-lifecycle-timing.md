# TCP：慢开始、连接建立与主动关闭计时

## 一条 TCP 连接的计时骨架

TCP 全流程题应先按“建立—传输—释放—等待”切段，再判断每段的计时终点。主动关闭方发出最后一个 ACK 后只是进入 TIME_WAIT，还要等待两倍最长报文段寿命（Maximum Segment Lifetime，MSL）才进入 CLOSED。

```text
主动端 H                                           服务器 S
   │──── SYN ────►│
   │◄── SYN+ACK ──│  建立连接：1 RTT
   │──── ACK ────►│  ACK 可携带首批数据
   │               │
   │══ 第 1 轮数据 ═►│  初始 cwnd = 1 MSS
   │◄════ ACK ═════│  1 RTT 后 cwnd 增至 2 MSS
   │══ 第 2 轮数据 ═►│  可发送 2 MSS
   │◄════ ACK ═════│
   │               │
   │──── FIN ────►│
   │◄──── ACK ────│
   │◄──── FIN ────│  最快 1 RTT 后收到 FIN
   │──── ACK ────►│  进入 TIME_WAIT
   │··············│  等待 2 MSL
   ▼    CLOSED     │
```

## 数据要经过几轮 RTT

最大报文段长度（Maximum Segment Size，MSS）限制每个 TCP 段携带的应用数据量。报文长度为 `L` 时，段数为：

```text
N = ⌈L / MSS⌉
```

408 的慢开始计算题若未另给初始值，通常按初始拥塞窗口 `cwnd=1 MSS` 计算。每轮所有段都被确认后，`cwnd` 按 `1, 2, 4, 8, ... MSS` 增长；第 `k` 轮累计最多发出：

```text
1 + 2 + 4 + ... + 2^(k-1) = 2^k - 1 个 MSS 段
```

因此找到满足 `2^k-1 ≥ N` 的最小 `k`，就是无丢包、接收窗口不构成限制时的数据传输 RTT 数。若题目明确给出初始拥塞窗口、接收窗口或确认规则，应以题设为准，不能套用默认的 `1 MSS`。

## 建立与释放的边界

- 主动端发出 SYN，到收到 SYN+ACK 后即可用第三次握手的 ACK 携带数据，按 `1 RTT` 计建立阶段。
- 主动端发出 FIN 后，服务器对 FIN 的 ACK 与服务器自己的 FIN 可以连续发出；主动端最快在 `1 RTT` 后收到 FIN、发出最后一个 ACK 并进入 TIME_WAIT。
- TIME_WAIT 的长度是 `2 MSL`，不是 `1 MSL`。它既允许最后一个 ACK 丢失后再次确认重传的 FIN，也让旧连接中的迟到报文段在网络中消失。
- 题目若问主动关闭方何时进入 CLOSED，必须加 `2 MSL`；若只问何时进入 TIME_WAIT，则不加这段等待。

忽略 TCP 段传输时延时，一道完整计时题可写成：

```text
总时间 = 建立连接 RTT 数 × RTT
       + 数据传输轮数 × RTT
       + 主动关闭至 TIME_WAIT 的 RTT 数 × RTT
       + 2 MSL
```
