# VLAN：广播域与 ARP 可见范围

## VLAN 怎样划分广播域

虚拟局域网（Virtual LAN，VLAN）把一台交换机上的端口分成多个逻辑局域网。属于同一 VLAN 的端口处在同一个二层广播域；属于不同 VLAN 的端口即使接在同一台交换机上，也不能直接交换二层广播帧。

```text
VLAN1                                VLAN2             VLAN3
H1   H2   H3   H4                   H5                H6   H7
          ◄── ARP 广播 ── H4        │                 │    │
════════════ 广播域边界 ════════════╪═════════════════╪════╪══
交换机只在 VLAN1 的端口内泛洪，不把该广播送入 VLAN2、VLAN3
```

交换机转发广播帧时，仍先看帧所属的 VLAN，再只向该 VLAN 内除入端口外的其他端口泛洪。VLAN 因而缩小了广播范围；物理上接在同一交换机，不等于处在同一广播域。

## ARP 请求能到达谁

地址解析协议（Address Resolution Protocol，ARP）用目标 IP 地址查询本地链路上的 MAC 地址。主机不知道目标 MAC 地址时，发送目的 MAC 地址为 `FF-FF-FF-FF-FF-FF` 的 ARP 请求；目标主机收到后返回自己的 MAC 地址，通信双方可以把 IP—MAC 对应关系写入 ARP 表。

ARP 请求是二层广播，因此不能越过 VLAN 的广播域边界：

- 同一 VLAN 内的主机能够收到请求，其映射可能出现在彼此的 ARP 表中；
- 不同 VLAN 内的主机收不到请求，不能通过这次 ARP 直接学习对方的 MAC 地址；
- 访问其他网络时，主机解析的是默认网关接口的 MAC 地址，再由三层设备转发，不会解析远端主机的 MAC 地址。

IP 地址前缀相同也不能突破 VLAN 边界。若两台主机被配置成同一 IP 子网却分属不同 VLAN，它们会误以为对方在本地链路并直接发送 ARP；广播无法跨 VLAN，通信仍然失败。
