# VLAN 与 ARP：谁的 IP—MAC 能被本机学到

## 一台交换机为什么能有多个局域网

问题的关键不是主机是否插在同一台交换机上，而是它的端口属于哪个 VLAN。虚拟局域网（Virtual LAN，VLAN）给交换机端口分组；每一组都是独立的二层广播域。交换机收到广播帧后，先确定该帧所属的 VLAN，只在这一组中、除入端口外的端口泛洪。

```text
                         一台交换机
VLAN1：H1、H2、H3、H4       VLAN2：H5       VLAN3：H6、H7

本机 ── ARP 广播 ──► 同一 VLAN 的其他主机
 │                    ▲
 └── 只在 VLAN1 泛洪 ──┘     × 不会进入 VLAN2、VLAN3
```

因此，“物理上同接一台交换机”不能推出“二层上可以互相看见”。VLAN 的边界就是广播帧不能直接穿过的边界；若需要在 VLAN 间通信，必须由三层设备转发。

## ARP 表项怎样产生

地址解析协议（Address Resolution Protocol，ARP）解决的是同一条本地链路上的问题：已知目标 IP 地址，怎样得到应写入以太网帧的目标 MAC 地址。

```text
本机不知道目标 MAC
        │
        ▼
ARP 请求：目标 IP 是谁？目的 MAC = FF-FF-FF-FF-FF-FF
        │             （二层广播）
        ▼
同一 VLAN 的目标主机回答自己的 MAC ──► 本机记录 IP ─ MAC ──► 以后单播
```

ARP 请求是广播，所以它能到达谁，完全由 VLAN 的广播域决定。本机只能经 ARP 学到同一 VLAN 中主机的 IP—MAC 对应；其他 VLAN 中的主机收不到请求，本机不可能通过 ARP 得到它们的 MAC 地址。

ARP 表中的 TTL 只表示已经学到的表项还可保留多久。TTL 不会让请求越过 VLAN，也不能把一个从未收到的应答变成表项。

## 遇到异 VLAN 的 IP 地址时解析谁

主机先按自己的 IP 配置判断目标是否在本地网络。若目标在其他网络，本机仍然要发送以太网帧，但它查询的不是远端主机的 MAC，而是本 VLAN 上默认网关接口的 MAC：

```text
本机 ── ARP 查询网关 MAC ──► 本 VLAN 上的网关 ── 三层转发 ──► 目标 VLAN
```

所以，两个主机的 IP 地址前缀相同，也不能消除 VLAN 边界。若它们被错误地配在同一 IP 子网却分属不同 VLAN，发送方会误把对方当作本地目标并广播 ARP；广播到不了对方，通信仍会失败。
