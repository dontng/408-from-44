# 以太网 CSMA/CD：竞争、冲突与二进制指数退避

## 碰撞怎样发生

CSMA/CD 用于**共享、半双工**以太网：多台主机共用一条链路，同一时刻只能有一台成功发送。

- **载波监听（CS）**：发送前先听；忙就等，空闲就发。
- **多点接入（MA）**：A、B 相距较远时，A 的信号还没到 B，B 仍会听到空闲并发送；两路信号相遇便碰撞。
- **冲突检测（CD）**：发送时仍继续监听。收到的信号与自己正在发送的不一致，就说明发生碰撞；立即停止本帧，随机等待后再重发。

## 争用期

```text
A ───── τ ─────► B ───── τ ─────► A
        2τ：最远两站往返传播时间
```

`2τ` 就是最远两站 A→B→A 的往返传播时间；题目给单程距离时，才写 `τ = 距离 / 传播速度`。

帧发送时间必须 `≥ 2τ`，否则帧发完才收到碰撞信息。以太网把争用期规定为 **512 个比特时间**：1 比特时间就是发送 1 bit 所需时间；速率为 `R b/s` 时，争用期为 `512 / R s`。网络必须满足 `2τ ≤ 512 / R s`。

最小帧为 `512 b = 64 B`，所以最小帧的发送时间恰好覆盖一个争用期。`b` 是 bit，`B = 8 b`，`1 Mb = 10^6 b`。

- 10 Mb/s：争用期 = `512 / (10×10^6) s = 51.2 μs`。
- 100 Mb/s：争用期 = `5.12 μs`。
- 传播速度 `200 m/μs`、距离 2500 m：`2τ = 2×2500 / 200 = 25 μs`；在 10 Mb/s 下小于 `51.2 μs`，可以检测碰撞。

## 二进制指数退避

碰撞后等待 `r × 争用期` 再重发；`r` 随机取，避免两台主机又同时重发。

- `1 ≤ k ≤ 10`：`r` 从 `0` 到 `2^k−1` 随机取；冲突越多，等待范围越大。
- `10 < k < 16`：`r` 从 `0` 到 `1023` 随机取；范围不再扩大。
- `k = 16`：丢弃该帧。

题目问最大等待时取 `r` 的右端。11 次冲突、10 Mb/s：`1023 × 51.2 μs = 52.3776 ms`。

`k` 按题干定义代入；若题干按重传次数计，不能自行加减一次。
