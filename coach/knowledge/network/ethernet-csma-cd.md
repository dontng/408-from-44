# 以太网 CSMA/CD：竞争、冲突与二进制指数退避

CSMA/CD 解决的是**多台主机共用一条、半双工以太网链路**时谁能发送的问题：同一时刻只能有一台成功发送。即使大家先监听，信号传播也需要时间；相距较远的两台主机仍可能同时认为链路空闲，于是发生碰撞。

## CS、MA、CD：一轮发送到底做了什么

### CS：发送前先听，空闲就立刻发

**载波监听**（Carrier Sense，CS）是发送前检测介质上有没有别人的信号。

- 介质忙：继续监听，不发送。
- 介质空闲：立即发送。这叫 **1-坚持**（1-persistent）：一旦听到空闲，就以概率 1 发送，不再额外等待。

1-坚持提高了链路利用率，但多台已经等候的主机可能在同一刻都听到空闲，也就在同一刻发送。

### MA：多台主机共享，监听也不能消除传播延迟

**多点接入**（Multiple Access，MA）指多台主机共用同一传输介质。设 A、B 相距较远：A 刚开始发送时，信号还在路上传播，尚未到 B；B 因而仍会听到“空闲”并开始发送。两路信号相遇后就是碰撞。

### CD：边发边听；碰撞后让所有站都知道

**冲突检测**（Collision Detection，CD）要求主机发送期间继续监听介质。检测到碰撞后：

1. 停止发送当前帧；
2. 发送 **Jam 信号**（强化碰撞的短信号），让碰撞状态持续得足够久，使链路上其他正在发送的主机也能发现这次碰撞；
3. 按二进制指数退避等待，再重新监听、重发。

CSMA/CD 只对应共享、半双工以太网；交换式全双工以太网没有这种共享碰撞域，不使用 CD。

## 争用期 `2τ`：最晚何时必须发现碰撞

```text
A 开始发送 ───── τ ─────► B
A 收到 B 的碰撞信号 ◄──── τ ───── B 开始发送
                         2τ：最远两站往返传播时间
```

`τ` 是最远两站的单程传播时间，`2τ` 是往返传播时间。若 B 在 A 的信号到达前发送，碰撞的信息最晚要经过一个往返才回到 A；因此 A 的帧还必须在发送，才能检测到碰撞。

- **争用期**：`2τ = 512 bit times`。
- **检测条件**：帧发送时间 `≥ 2τ`；以太网的最小帧长为 `512 b = 64 B`，正是为了覆盖这个窗口。
- 单位：`b` 是 bit，`B = 8 b`；`1 Mb = 10^6 b`；`1 s = 10^3 ms = 10^6 μs = 10^9 ns`。

### 从速率或距离代入

- 10 Mb/s = `10 b/μs`，故 1 bit time = `0.1 μs`，争用期 = `512 × 0.1 = 51.2 μs`。
- 100 Mb/s = `100 b/μs`，故 1 bit time = `0.01 μs`。
- 若传播速度为 `200 m/μs`，相距 2500 m 时：`τ = 12.5 μs`，`2τ = 25 μs`。
- 在 10 Mb/s 下，最小帧发送时间 `51.2 μs > 25 μs`，故该场景能检测碰撞。

## 二进制指数退避：碰撞后等多久

碰撞后随机等待 `r × 争用期` 再重发；随机而不是固定等待，是为了避免刚发生碰撞的两台主机又同时重发。

| 已发生冲突次数 `k` | 随机数 `r` | 处理 |
| --- | --- | --- |
| `1 ≤ k ≤ 10` | `0` 到 `2^k−1` | 随冲突增加，候选等待时间加倍 |
| `10 < k < 16` | `0` 到 `1023` | 窗口封顶 |
| `k = 16` | — | 丢弃该帧，不再重传 |

题目问最大等待时间时取 `r` 的右端。例如 11 次冲突、10 Mb/s：`r = 1023`，最大等待为 `1023 × 51.2 μs = 52.3776 ms`。

`k` 必须按题干定义：若题干按“已发生 `k` 次冲突”计，直接代入；若按重传次数计，不能自行多加或少减一次。
