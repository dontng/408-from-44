# 以太网 CSMA/CD：二进制指数退避

在半双工共享以太网中，站点检测到冲突后不能立刻重发，而是等待随机的退避时间：

```text
退避时间 = r × 争用期
r ∈ [0, 2^min(k, 10) − 1]
```

其中 `k` 是本帧已发生的冲突次数；争用期为 `512 bit times`。因此当题目问“第 `k` 次冲突后再次发送的最大时间间隔”，取 `r` 的上界。指数从第 10 次冲突起封顶，不能继续按 `2^k` 增长。

对 10 Mb/s 的以太网，一个比特时间是 `0.1 μs`，所以争用期为 `512 × 0.1 μs = 51.2 μs`。例如第 11 次冲突后，最大退避为：

```text
(2^10 − 1) × 51.2 μs = 52.3776 ms
```

不要把“连续发生 k 次冲突”误读为 `k` 个争用期，也不要把第 11 次继续代入 `2^11`。
