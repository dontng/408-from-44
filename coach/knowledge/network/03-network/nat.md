# NAT：地址与端口转换、UDP 校验和

## 地址和端口怎样转换

NAT 使使用私有 IP 地址的主机能够访问 Internet。多台内网主机共用一个公网 IP 地址时，路由器还要用端口号区分不同通信；408 将这种地址和端口一起转换的方式也统称为 NAT。

```text
内网主机                    NAT 路由器                         Internet 服务器
192.168.1.5:49152 ──► 公网 IP:40001 ⇄ 192.168.1.5:49152 ──► 服务器 IP:53
                        改源 IP、源端口                    目的 IP、目的端口不改
```

路由器把内网的“IP 地址、端口号”换成公网的“IP 地址、端口号”，并把对应关系记在 NAT 表中。返回报文到达公网端口 `40001` 后，路由器查表，才能把它交还给 `192.168.1.5:49152`。服务器及其应用没有改变，所以目的 IP 地址和目的端口号不变。

## UDP 总长度为什么不变

UDP 首部固定为 8 B，源端口、目的端口、总长度和校验和各占 2 B。NAT 只替换源端口字段中的数值，并没有给 UDP 报文段增添字节，因此：

```text
UDP 总长度 = 8 B 首部 + 数据
```

这个长度不因 NAT 而改变。路由器转发时会去掉原链路层首尾，再为下一段链路重新封装；链路层首尾不属于 UDP 报文段，也不计入 UDP 总长度。

## UDP 校验和为什么改变

UDP 校验和的计算范围不只有 UDP 首部和数据，还包括由 IP 层信息组成的伪首部：

```text
UDP 校验和
├─ 伪首部：源 IP、目的 IP、协议号、UDP 长度
├─ UDP 首部：源端口、目的端口、UDP 长度、校验和
└─ 数据
```

伪首部只参加计算，不会随 UDP 报文段发送，也不会增加 UDP 总长度。NAT 改了 UDP 源端口，也改了伪首部中的源 IP 地址；原校验和已经不再对应转换后的内容，必须随之更新。
