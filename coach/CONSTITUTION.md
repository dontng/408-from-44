# Coach Constitution

Coach 是 408 真题学习资产的根。它把一题的真实处理结果交付给三个实体层，并让每日 notes 呈现当天的最终产出。

## Stable layout

```text
coach/
  CONSTITUTION.md
  ENTRY.md
  current.md
  today/
  knowledge/
    network/
      00-media-infrastructure/
      01-physical/
      02-data-link/
      03-network/
      04-transport/
      05-application/
  ability/
  speedrun/
  analysis/
  notes/<month>/<MMDD-dayXX-note>.md
```

- `knowledge/` 是唯一的 408 学科正文。它按科目与章节生长，写成可持续修订的教材，不按年份堆叠。
- `analysis/` 按 `<year>/qNN.md` 保存原题的硬解析。正文直接处理题干、推理与选项；它调用 knowledge，不复制一份课程。
- `ability/` 保存已经由真实题目证实的个人解题能力内容。它不重讲学科知识，也不预建分类树。
- `speedrun/` 保存可在考场调用的**决策入口**。一页只压缩一个稳定的题干→判断机制；它链接 knowledge 的原理与 analysis 的题证，不重复两者，也不接管题单。
- `notes/` 是下游呈现。目录必须与 `src/` 一致，按月份管理；每日 note 引用当天题目沉淀出的 analysis、knowledge、ability，不复制它们的正文。
- `today/` 与 `current.md` 仍是 T0 到 coach 的运行入口。

计算机网络知识按国内 408 教学使用的五层体系归档，并在物理层之前保留非正规的“第 0 层”。数字固定自底向上的教学顺序，不依赖英文名称的字母排序：

- `00-media-infrastructure/`：非正规第 0 层，保存传输介质和基础设施环境，如双绞线、同轴电缆、光纤与无线介质；介质承载信号，但本身不执行物理层协议功能；
- `01-physical/`：第 1 层，物理层，保存接口特性、信号、编码与调制、比特率与码元速率、复用等机制；
- `02-data-link/`：第 2 层，数据链路层；
- `03-network/`：第 3 层，网络层；
- `04-transport/`：第 4 层，运输层；
- `05-application/`：第 5 层，应用层。

目录编号是教学索引，不改写正式分层：物理层仍是第 1 层，“第 0 层”只是不属于 OSI 层次的非正规称呼。遇到用户用语与正式模型表面冲突时，必须先辨析它指向的机制与边界，不能按字面直接建立目录。

知识页按它主要解释的机制归档。跨层知识不拆成多份：例如 NAT 即使涉及 UDP 端口和校验和，主要机制仍是网络层路由器的地址转换，因此归入 `03-network/`，正文再解释必要的跨层影响。

## Content ownership

一段正文只能有一个主要归宿：

| 内容 | 归宿 |
| --- | --- |
| 脱离原题后仍成立的学科解释 | knowledge |
| 某题如何被直接做出、如何逐项排除选项 | analysis |
| 用户为何未能正确调用知识与解析路径，以及怎样训练 | ability |
| 一类题出现时应抓哪条条件、怎样最短地判断或排除 | speedrun |
| 当天完成了什么、这些实体之间的引用 | notes |

## Knowledge: mechanism, not summary

knowledge 是可检索、可复用的 408 教材正文，用来替代重新听一段课和重新整理一次笔记。它必须使相关真题能够从术语、机制到计算或判断被重新调用；不能只记录结论、题目答案或抽象公式。

每个 knowledge 实体的首屏必须包含一幅机制图；图应紧邻它解释的概念，不强制占据文件最顶端。图是正文：使用代码区中的空间关系表达对象、时间、层级、因果或计算路径；标题、列表、表格和公式只能辅助图，不能替代图。图必须先把机制讲清，再压缩版面；不得为了统一样式使用 Mermaid 流程图、超大图片或空白很多的卡片。

压缩的单位是重复关系，不是必要解释。相同规则覆盖的多个情况应合并为一个区间或分支；术语定义、发生条件、单位换算、边界值和关键因果不得删去。正文应保留正式中文名、英文名/缩写、符号、单位和关键数值，使其可被 Ctrl+F 直接定位。

### Explanation standard: textbook prose, not term notes

knowledge 要像写成定稿的经典教材，不像术语笔记或作者的思考记录。先用最短的中文把现象、机制和结论说通，再给名词、缩写和符号作标签；术语不能替代解释，也不能为了显得专业而出现。

作者必须主动追求**完整而不费力**的表达：每句只做一件事，每段只推进一个问题；不参与本页推理的事实、比较和名词删除。新术语若不能立刻用一句自然中文说明它是什么、为什么现在需要它，就先不写。表格、图和公式只在它们比一句话更清楚时使用。

读不通或被指出不清楚时，回到“读者先看到什么问题、随后凭什么得出结论”重写整个相关段落；不得在旧句上叠加术语、括号、例外和补充说明。定稿前应能让不预先知道缩写的读者一路读到结论，也应让熟悉该章的读者看不见多余的解释。

公式不得孤立出现。每个公式相关量必须在同一视觉块中交代其来源、物理意义、单位变化和题目中的代入路径。若题目可能从距离、速率、时间、帧长或次数任一入口切入，knowledge 必须把这些入口连接到同一机制，而不是只保留其中一种算式。

“一屏”指 VS Code 默认编辑区左右分栏后，其中一栏可见的有效编辑区域。首屏应在这个宽度与高度内尽可能展示完整骨架、机制图与高频入口；首屏以下允许保留不能安全压缩的推导、换算、边界或变体。禁止把“一屏”误用为省略关键知识的理由。analysis 只引用 knowledge 并处理原题，不能把题目的数值例子和选项判断倒灌成 knowledge 正文。

### Generation method: close the concept graph before writing

knowledge 的固定根基是问题所需的概念及其真实关系，不是行文感觉。写作决策按以下次序服从：事实和关系正确，解释闭合，融入原文，凝练，最后才是标题层级、图表位置和首屏密度等形式。后一项不得损害前一项；形式上更整齐但更难懂，就是劣化。

新增或修订前，先在内部建立关键概念关系网，不直接攒正文：

1. 用一句话确定本次只需要讲清的中心问题。
2. 提取回答它不可缺少的关键概念，并为同一概念固定一个名称。
3. 用最凝练、简单、直白的话解释每个概念。解释若用了读者还不懂的概念，就把新概念加入关系网继续解释，直到只依赖常识和前文已经说明的内容。
4. 只连接真实存在的因果、组成、区别、换算、条件和边界。每条连接都必须能说成一句有依据的话；不能连接的概念应分开处理或删除，不能为了连贯硬接。
5. 检查关系网是否闭合：没有未解释的概念，没有孤立概念，没有跳过的推理，数字和单位都有来源。读者应能沿这些连接暂时复述出正确机制，而不只是记住结论。

关系网只是生成骨架，不能原样写成术语卡片。闭合后再按概念的依赖顺序翻译成自然正文：第一次用到概念时就解释，后文沿用同一名称；一句只承担一个连接，一段只回答一个局部问题。机制图只承载比文字更适合用空间关系表达的部分，不能抢在必要解释之前；表和公式也不能代替直白说明。权威文档只负责校验事实和边界，不能代替解释。

修改既有实体时，先读完相关页面，优先复用它已有的概念名称和解释顺序。若 probe 暴露的缺口仍属于原页的中心问题，就重写能独立读通的最小连续段落；只有出现独立的中心问题、稳定的检索入口和自己的概念关系网时，才新建 knowledge。不得用“补充”“需要注意”、括号或孤立段落把新内容粘在旧文上，也不得让 probe 保存一份平行正文。

交付前脱离补丁视角连续读完整个相关小节，并逐句反查：每句话对应哪个必要概念或真实连接，删掉后是否影响理解。没有对应项的句子删除。只有当一个原本不懂的人能够沿正文建立正确理解，并且全文没有重复、矛盾或明显接缝时，才能交付；不能等待用户指出缺项后再补。

## Pass

`pass` 是一题学习过程的收束与交付，不是一个独立文件或目录，也不要求把一切问题当场解决。

一次 pass 至少保证：本题已经被理解；有价值内容不只留在聊天上下文或原卷空白处；它已被交付给既有或待创建的 knowledge、analysis、ability 实体，并可由当天 note 呈现。

`revisit` 与 `pin` 目前只保留为未来可实现的关系标记。不得为它们新建目录、队列或独立知识正文。

## Analysis rule

analysis 不写“题目档案式”小作文。打开文件后直接解题：能正推就正推，正推成本高时就逐项排除；两种方法都必须调用已有 knowledge。若 knowledge 不足以把题做穿，应补 knowledge 后回到 analysis。

## Speedrun rule

speedrun 不是新的知识库、错题本或每日任务表。题单仍由 `today/` 与 `current.md` 决定；knowledge 仍负责把概念讲懂；analysis 仍负责做穿原题。speedrun 只回答一个考场问题：**看到什么题干条件时，用什么最短的判断动作，能分开哪些选项。**

新页先是 `candidate`，不能算作已获得的通关能力，也不能因为写完就从题单中消失。要升为 `verified`，至少要有两个实际题目或一道实际题目与一道记录在页内的改条件检验；遮住答案后能复述决策轨迹，并在未见或改条件题中保持正确。若只适用于单题，留在 analysis，不新建 speedrun。

每页固定写清五件事：适用边界、最小前提、决策动作、干扰项机制、证据与验收。所谓“最小前提”必须相对于选项差异定义——它是区分真伪所必需的不变量、条件或反例，不是把一段讲义缩短。

## Context and quality

- 当前题只读取必要的 knowledge 段落、相关 analysis 与少量已证实的 ability 内容；不得批量读取全库。
- 不因措辞漂亮、看似提速或一次偶然失误新建长期内容。
- 所有关于用户的判断必须可回链到真实题目，并允许后续题推翻。
- 任何新增结构必须服务下一道相关题更快、更准地被处理；否则不保留。
