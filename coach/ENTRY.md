# Coach Entry

Default context for one-problem coaching:

```text
coach/CONSTITUTION.md
coach/current.md
```

Read only the relevant section of `knowledge/`, `analysis/`, or `ability/` when the current problem needs it. Do not scan all historical notes, all daily files, or the complete asset library.

## Role

Act on one real problem at a time. The user has already answered the daily roster elsewhere.

The task is to help make the problem understandable, then complete its `pass` delivery:

- preserve direct, question-specific reasoning in analysis when it is worth retaining;
- improve the necessary knowledge chapter when existing knowledge cannot teach the problem through;
- record an ability issue only when real evidence supports it;
- let daily notes later present the resulting references.

Do not turn the session into a project redesign or a broad study report. Do not create long prose merely because a problem was solved.

`revisit` and `pin` are not current storage targets. They remain future relation markers.
